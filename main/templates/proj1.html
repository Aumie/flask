People Counter

         This Project is about counting people when they are entering/exiting in an area.In a detection part, I use MobileNet-SSD Caffe implementation and OpenCV, and then this will process any frame which is passed by IP camara or webcam for sending to the GUI (which is programmed with PyQt5). At this part, I got stuck at the performance of projecting frames, it had a lot of delay, so I used a trick that is called "multi-thread programing". It solved this problem easily (I think I don't understand it much to write it in the larger system, but if I have to do it, I'll put more effort to gain this power.), moreover it helped me a lot in displaying the loading status.

         GUI section, I designed it to have 1 main window for interacting with other camera (just only for filling the ip and opening camera.) and 4 camera's windows for observing the result from each camera which is able to specify the limit of people to be left in an area. If the number reaches the limit, it notifies users in the line group. And last one, Database, I chose firebase from google, It is very handy from my view.

         After all, I think this program is not good enough, it still leaves some bugs behind in some cases that I don't realize. From this project, I didn't step into those model(e.g. Training a model from TensorFlow to do something) much. So, it gave me a curiosity about what happened in the processing stage of those frames and it motivates me, so I have started to read about them from this book.

Tools I (really) used :
  • OpenCv
  • Firebase
  • Line notify
  • PyQt5
  • Git
  • Qt Designer
This project is pure python :D
